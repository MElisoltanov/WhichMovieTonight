# Results Summary and Lessons Learned


# Final Report: WhichMovieTonight

---

## 1. Results Summary

### Core MVP Features
- Complete user journey: registration, login, browsing the movie catalog, viewing details, rating and commenting, and displaying streaming availability.
- Secure authentication (JWT), permission management (admin/user), and React/Django REST API integration.
- Mobile-first, responsive interface with search and filters.

### Comparison with Initial Objectives
- Most of the features planned in the Project Charter were delivered:
  - Movie catalog with our own database
  - User accounts, ratings, comments
  - Streaming availability information
- Some secondary features (favorites, badges, chat) were considered out of MVP scope.
- Sprint deadlines were generally met, with a stable and tested MVP delivered.

### Key Metrics
- 90% of MVP user stories completed
- Application tested on multiple browsers and devices
- Positive user feedback on usability during testing

---

## 2. Lessons Learned

### What Went Well
- Effective daily communication (WhatsApp, short meetings)
- Clear division of backend/frontend tasks
- Fast adaptation to new tools (Django, React, JWT)
- Regular integration testing, real-time bug fixes

### Challenges Faced
- Learning curve with Django REST and JWT management
- Lack of formal tracking tools (no Trello/Jira)

### Areas for Improvement
- Implement a project management tool for better traceability
- Allocate more time for user testing and documentation

---

## 3. Team Retrospective

### What Worked Well as a Team
Moussa and I had already collaborated on several projects during our Holberton journey, which made teamwork easier. We communicated regularly and simply about every issue or question encountered, which greatly facilitated the progress of the project.

### Challenges Faced and How They Were Resolved
The beginning was inevitably difficult: setting up the structure, dependencies, and understanding how new technologies work. Discussing and exchanging ideas together helped us face these challenges. Step by step, we consulted documentation, tried to understand through videos we recommended to each other, and then, in a second phase, we also used AI to move forward in a guided and efficient way given the limited time. During the MVP, we followed an informal plan and communicated regularly. This allowed us not to get lost and to refine the structure of our MVP for a clear result.

### Suggestions to Improve Collaboration in the Future
Create a more formal organization and a more professional management of GitHub and branches. We had a fairly informal organization for this project, which was compatible with the size of our team. To avoid getting lost in branches, we decided to create only two (frontend and backend) plus a dev branch. In the future, feature branches would be more interesting and professionalizing.

---

## 4. Conclusion

The WhichMovieTonight project achieved nearly all set objectives, while strengthening the team’s technical and organizational skills. Direct communication was key success factors. For future projects, a more structured Agile-based organization could be beneficial, even though the small team size allowed for an informal but effective approach.
